# 命宫 / 身宫公式审计（Phase 3）

> V1.2.6 Classical Calibration — 先记录差异，再决定是否改码。

## 当前引擎公式（`palace_engine.py`）

### 命宫

```
month_palace = (2 + lunar_month - 1) % 12   # 寅起正月
ming_gong    = (month_palace - hour_branch_index + 12) % 12
```

口诀实现：**寅起正月，顺数至生月，再逆数至生时**。

### 身宫

```
shen_gong = (month_palace + hour_branch_index) % 12
```

口诀实现：**寅起正月，顺数至生月，再顺数至生时**。

## 传统三合派公式

与上式一致（地支索引：子=0 … 亥=11；时辰同索引）。

## 示例（REF-01 / SC-C01）

农历四月、未时(index=7)：

与 SC-C01 reference 一致。

## 可能导致与文墨天机不一致的外部因素（非本公式错误）

1. **真太阳时**改变时辰 → 命宫整体平移  
2. **农历/公历**输入混用  
3. **早/晚子时**换日规则  
4. **节气月 vs 农历月**安命（本引擎用农历月）

本阶段：**不修改** `palace_engine.py` 命身宫公式（与经典及 SC-C01 一致）。
